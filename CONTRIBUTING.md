# Contributing to Inst-electronica-p2-interrupciones

Thank you for your interest in contributing to the Inst-electronica-p2-interrupciones repository! We welcome contributions, and your input is invaluable. Please review the guidelines below to get started.

## Reporting Issues

1. **Check Existing Issues**: Before creating a new issue, please check the existing issues to see if someone else has already reported it or if it is being worked on.
2. **Provide Details**: When reporting an issue, please provide as many details as possible, including:
   - A clear and descriptive title
   - A description of the issue and steps to reproduce it
   - The expected behavior and what actually happened
   - Any relevant screenshots or error messages

## Submitting Pull Requests

1. **Fork the Repository**: Create a fork of the repository on GitHub.
2. **Create a Branch**: Create a new branch in your fork to work on your changes. Try to use a descriptive name for your branch related to the issue or feature you are addressing.
3. **Make Your Changes**: Implement your changes and ensure everything is functioning correctly. Please follow the project's coding style and conventions.
4. **Commit Your Changes**: Write clear and concise commit messages that describe the changes you’ve made. Use present tense and imperative mood.
5. **Submit the Pull Request**: Navigate to the original repository on GitHub, and submit a pull request from your fork. Include a description of your changes and reference any related issues.

## General Contribution Rules

- **Code of Conduct**: Please adhere to our code of conduct and be respectful to other contributors.
- **Testing**: Ensure that all new code is tested and passes existing tests. If you are adding new features, consider adding tests for them as well.
- **Documentation**: Update any relevant documentation when making changes to the codebase.

Thank you for considering contributing to our project! We look forward to your contributions.
